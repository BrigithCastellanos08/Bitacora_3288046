PROYECTO FINAL DE FUNDAMENTOS DE SOFTWARE

Algoritmo

Secuencia finita y ordenada de intrucciones, logicas y precisas que se siguen paso a paso para resolver un problema especifico,realizar una tarea o lograr un objetivo. Actua como un diseño previo a la escritura del codigo

Programa

Es un conjunto ordenado de instrucciones escritas en un lenguaje de programación que una computadora interpreta para ejecutar tareas específicas o resolver problemas. Estos códigos son traducidos a un formato que la máquina pueda entender y ejecutar

Codigo Fuente

Es el conjunto de instrucciones escritas por un programador en un lenguaje de programación, que sirven como el "plano" para un programa de computadora. Este código, que no es directamente ejecutable por la máquina, se traduce a un lenguaje de máquina (código binario) mediante un compilador o intérprete

Compilacion

Es el proceso de traducir el código fuente, escrito en un lenguaje de alto nivel (legible por humanos), a un lenguaje de bajo nivel (código máquina o binario), que la computadora pueda entender y ejecutar directamente. Este proceso lo realiza un programa llamado compilador, cuyo resultado suele ser un archivo ejecutable

Interprete

Es un programa que lee, traduce y ejecuta el código fuente de un lenguaje de alto nivel, instrucción por instrucción, de forma inmediata y sobre la marcha, sin generar un archivo ejecutable final, a diferencia de un compilador que traduce todo el código a la vez antes de su ejecución.

Binario

Es el lenguaje fundamental de las computadoras, que utiliza los dígitos 0 y 1 para representar datos e instrucciones, permitiendo que los dispositivos procesen y almacenen información como texto, imágenes y sonido. Cada combinación de 0s y 1s, llamada bit, puede representar un número, carácter o instrucción diferente

Framework

Es un conjunto estandarizado de herramientas, componentes y reglas que proporciona una estructura base para el desarrollo de aplicaciones de software, acelerando el proceso al ofrecer soluciones a problemas comunes, promoviendo buenas prácticas y la reutilización de código

Libreria

Son colecciones de código reutilizable (funciones, clases, etc.) que los desarrolladores utilizan para realizar tareas comunes sin tener que escribir el código desde cero. Ayudan a agilizar el desarrollo, reducir la duplicidad de código, minimizar errores y disminuir costos

Script

Es un conjunto de instrucciones escritas en un lenguaje de scripting que se interpreta directamente por un motor o intérprete para automatizar tareas, manipular sistemas existentes o personalizar la funcionalidad de un programa. A diferencia del código compilado, los scripts se ejecutan sin un paso de compilación previo, lo que los hace ideales para tareas rápidas, personalización y automatización en tiempo real

Codigo

Es un conjunto de instrucciones escritas en un lenguaje de programación específico (como Python, Java o C++) que le indica a un ordenador qué tareas debe realizar. Este código se organiza en un archivo fuente, se traduce a código máquina para ser ejecutable y es la base para crear software, aplicaciones y sistemas informáticos.

Ingieneria Software

Es la aplicación sistemática de principios de ingeniería para el diseño, desarrollo, mantenimiento y evolución de software de alta calidad, confiable y eficiente. Mientras que la programación es el proceso de escribir el código, la ingeniería de software es el marco completo que abarca todo el ciclo de vida del proyecto, desde el análisis de requisitos hasta las pruebas y la gestión.

Requerimientos Funcionales

Es una descripción específica de la tarea o función que un sistema debe realizar para satisfacer un objetivo o necesidad del usuario. Estos definen qué debe hacer el sistema, incluyendo sus entradas, salidas y los comportamientos que debe exhibir. Son las acciones, características e interacciones que permiten a los usuarios alcanzar sus metas utilizando el sistema.

Requerimientos No Funcionales

n los atributos y restricciones de calidad que definen cómo opera el sistema, en lugar de qué funciones realiza. Establecen criterios sobre el rendimiento, la seguridad, la usabilidad y la portabilidad, entre otros aspectos, asegurando la eficacia, la estabilidad y la satisfacción del usuario.

Arquitectura de software

Es la estructura de alto nivel de un sistema, definiendo cómo se organizan sus componentes y cómo interactúan entre sí para cumplir requisitos funcionales y no funcionales como el rendimiento o la escalabilidad. Es el plano maestro de un sistema, que guía las decisiones técnicas y tecnológicas, usando patrones y principios para asegurar la calidad, mantenibilidad y eficiencia del producto.

Ciclo de vida del software

Es un proceso estratégico y un mapa de fases que guían la creación de un producto de software, desde la concepción hasta el mantenimiento, buscando la eficiencia y calidad. Las fases típicas incluyen planificación (necesidades y viabilidad), análisis de requisitos, diseño (arquitectura y especificaciones), desarrollo (codificación), pruebas (funcionalidad y corrección de errores), implementación (despliegue a usuarios) y mantenimiento (actualizaciones y soporte continuo)

El modelo en cascada

Es un proceso de desarrollo de software que organiza las fases del proyecto de forma lineal y secuencial, como el agua que cae en una cascada. Cada fase se completa antes de iniciar la siguiente, lo que significa que el progreso fluye principalmente en una sola dirección. Este enfoque se caracteriza por su rigidez y falta de flexibilidad, ya que es difícil hacer cambios en las fases previas una vez que se han finalizado y se ha avanzado al siguiente paso.

El modelo incremental

Es una metodología de desarrollo de software que divide un proyecto en partes pequeñas y manejables llamadas incrementos. Cada incremento añade nuevas funcionalidades al sistema existente, se desarrolla a través de ciclos iterativos y se entrega al cliente en versiones funcionales. Esto permite a los equipos ofrecer valor rápidamente, adaptarse a cambios en los requisitos y recibir retroalimentación temprana, reduciendo los costos y riesgos iniciales en comparación con el desarrollo de un sistema completo de una sola vez

El modelo interactivo

Puede referirse a varias cosas: la Programación Interactiva (escribir código mientras el programa está activo), el Mode Interactivo para análisis de datos (vista inmediata de resultados y visualizaciones), o la capacidad de un software de tener Elementos Interactivos (botones con los que el usuario interactúa). También puede significar un sistema donde el usuario interactúa con datos provisionales para refinar un modelo antes de su generación final, como en el modelado interactivo de SPSS Modeler.

Caso de uso

Describe una funcionalidad específica de un sistema desde la perspectiva de un usuario (actor), mostrando cómo este interactúa con el sistema para lograr un objetivo. Los casos de uso capturan los requisitos funcionales, definiendo el comportamiento básico y excepcional del sistema, y se documentan mediante una representación gráfica (Diagrama de Casos de Uso) y una descripción textual en formato de plantilla

UML

Es un lenguaje estándar de modelado visual, no un lenguaje de programación, que utiliza diagramas y notación gráfica para representar, visualizar y documentar el diseño y la arquitectura de sistemas de software. Sirve para entender y comunicar los componentes, estructuras y comportamientos de sistemas complejos desde diferentes perspectivas, como diagramas de clases o de casos de uso, apoyando así el análisis y diseño en las distintas fases del desarrollo

HTML

No es un lenguaje de programación, sino un lenguaje de marcado que se usa para estructurar el contenido de las páginas web mediante etiquetas y atributos, como párrafos , imágenes y enlaces . Se complementa con CSS para el estilo y JavaScript para la interactividad, siendo estos tres los pilares del desarrollo web.

CSS

Es un lenguaje de hojas de estilo en cascada para la programación web que se utiliza junto con HTML para controlar la apariencia y el diseño de las páginas web. Aunque su función principal es el diseño visual (color, tipografía, espaciado y posición de elementos), también permite crear elementos interactivos como animaciones y transformaciones, adaptándose a diferentes dispositivos. CSS separa el contenido (HTML) del estilo, mejorando la organización y el rendimiento de los sitios web

JavaScript

Es el lenguaje de scripting más popular del mundo, utilizado para crear páginas web interactivas y dinámicas, así como aplicaciones web, móviles y de escritorio. JavaScript se integra con HTML y CSS para formar la base de la web, permitiendo a los desarrolladores añadir funcionalidades como formularios, menús desplegables, animaciones y galerías de imágenes.

Frontend

Se refiere al desarrollo de la parte visible e interactiva de una aplicación web o sitio que el usuario final ve y con la que interactúa directamente, utilizando lenguajes como HTML, CSS y JavaScript para crear la estructura, el diseño, las animaciones y la funcionalidad del lado del cliente. Su objetivo es ofrecer una experiencia de usuario positiva, garantizando interfaces atractivas, responsivas y accesibles

Backend

Se refiere al desarrollo del lado del servidor de una aplicación, gestionando la lógica, las bases de datos y las APIs que no son visibles para el usuario final. Los programadores de backend utilizan lenguajes como Python, Java, Node.js o PHP para crear el código que permite a los usuarios interactuar con el servidor y las bases de datos, garantizando el funcionamiento y la seguridad de la aplicación.

Full Stack

Se refiere a la habilidad de un desarrollador para trabajar en todas las capas de una aplicación web, tanto en el frontend (lo que el usuario ve e interactúa) como en el backend (la lógica, bases de datos y servidores). Esto permite crear aplicaciones web completas de principio a fin, lo cual es muy valioso en el mercado laboral por la versatilidad y la capacidad de optimizar los equipos de desarrollo

Responsive design

Es un enfoque de programación que adapta automáticamente la disposición y el contenido de un sitio web para ofrecer una experiencia de usuario óptima en cualquier dispositivo (móvil, tableta, escritorio). Esto se logra utilizando HTML y, principalmente, CSS para crear un diseño con disposición fluida y elementos redimensionables, lo que mejora el SEO y la accesibilidad.

API

Es un conjunto de reglas y protocolos que permite a diferentes programas de software interactuar y comunicarse entre sí, intercambiando datos y funcionalidades de manera estandarizada. Funciona como un intermediario que facilita la integración de servicios externos en aplicaciones propias, sin necesidad de conocer los detalles internos de cada sistema, permitiendo la creación de aplicaciones más complejas y eficientes.

REST

Es un estilo de arquitectura de software que define un conjunto de reglas y restricciones para el diseño de interfaces de programación de aplicaciones (APIs) que se comunican a través de la web, buscando lograr confiabilidad, escalabilidad y eficiencia en sistemas distribuidos. En esencia, una API que sigue los principios de REST expone datos como recursos y utiliza métodos HTTP estándar (GET, POST, PUT, DELETE) para realizar operaciones sobre ellos

GraphQL

Es un lenguaje de consulta de código abierto para APIs que permite a los clientes solicitar los datos exactos que necesitan, con un único punto de entrada en lugar de múltiples puntos finales. Es una alternativa más flexible y eficiente a APIs REST, ya que evita la sobrecarga de datos al especificar la estructura de la respuesta deseada. Para su uso, un servidor implementa un esquema GraphQL que define la estructura de los datos, y los desarrolladores construyen consultas para interactuar con él

Python

Se refiere al uso de Python, un lenguaje de programación de alto nivel, interpretado y de propósito general, conocido por su facilidad de aprendizaje y sintaxis legible que facilita la escritura de código limpio y eficiente. Se utiliza en una amplia variedad de campos, como el desarrollo web, la ciencia de datos, el aprendizaje automático (IA) y la automatización de tareas. Su versatilidad se debe en gran parte a sus extensas bibliotecas, su naturaleza multiplataforma y una comunidad de desarrolladores activa.

Java

Se refiere al uso del lenguaje de programación Java, conocido por ser de propósito general, orientado a objetos y multiplataforma, lo que permite crear aplicaciones para diversos dispositivos y sistemas operativos. Java es ampliamente utilizado en el desarrollo de software empresarial, aplicaciones web, móviles y de macrodatos, destacando por su portabilidad ("Escribe una vez, ejecuta en cualquier lugar"), seguridad y robustez

C++

se refiere al uso del lenguaje de programación C++, un lenguaje compilado, de propósito general y multiparadigma, conocido por su eficiencia, velocidad y capacidad para trabajar a bajo nivel. C++ es una extensión del lenguaje C que incorpora la programación orientada a objetos, la programación genérica y la programación estructurada, siendo ideal para el desarrollo de sistemas operativos, videojuegos, software de alto rendimiento y aplicaciones que requieren control de recursos. Para empezar, se debe configurar un entorno de desarrollo con un compilador (como GCC o Visual C++) y un IDE (como Visual Studio)

C#

Es un lenguaje de programación moderno, de alto nivel y orientado a objetos, desarrollado por Microsoft y basado en la familia de lenguajes C. Se utiliza para crear una amplia gama de aplicaciones en diversas plataformas, incluyendo desarrollo web, aplicaciones móviles y de escritorio, servicios en la nube y juegos. Es conocido por su seguridad de tipos, facilidad de uso y productividad, y es un lenguaje de programación popular para el ecosistema .NET.

PHP

Es un lenguaje de programación de propósito general, de código abierto y ejecutado en el servidor, ideal para el desarrollo web, ya que permite crear sitios web dinámicos e interactivos. Se incrusta en código HTML y se integra con bases de datos para gestionar contenido de manera personalizada y eficiente, siendo una herramienta potente y flexible tanto para principiantes como para desarrolladores avanzados

Ruby

Es un lenguaje de programación de alto nivel, interpretado y de código abierto, inventado por Yukihiro Matsumoto en la década de 1990 con el objetivo de ser fácil de usar, productivo y divertido para el desarrollador. Destaca por su sintaxis elegante y natural, que facilita la lectura y escritura de código, es completamente orientado a objetos y se utiliza para diversas aplicaciones, incluyendo desarrollo web (a menudo con el framework Ruby on Rails), procesamiento de datos y análisis web.

Golang

Es un lenguaje de programación de código abierto desarrollado por Google en 2009, diseñado para ser simple, eficiente y escalable en la era de las máquinas en red y sistemas distribuidos. Combina la velocidad de lenguajes como C++ con la facilidad de uso de Python y JavaScript, destacando en la programación de sistemas, desarrollo web, y manejo de concurrencia.

Kotlin

Es un lenguaje de programación moderno, multiplataforma y de código abierto, desarrollado por JetBrains, diseñado para ser conciso, seguro e interoperable con Java. Es el lenguaje recomendado por Google para el desarrollo de aplicaciones Android, destacando por su productividad, claridad y una curva de aprendizaje ligera. Sus características principales incluyen tipado estático, interoperabilidad total con Java, soporte para null-safety y una sintaxis sencilla que reduce la cantidad de código necesario para la misma funcionalidad.

Swift

Es un lenguaje de programación moderno, seguro y potente, desarrollado por Apple, para crear aplicaciones para todas las plataformas de Apple (iOS, macOS, tvOS, watchOS), así como para servidores y la nube. Es conocido por su sintaxis clara y legible, su alto rendimiento y su capacidad para reducir errores, integrándose además con código Objective-C. Para empezar a programar en Swift, se pueden utilizar herramientas como Xcode, disponible en el Mac App Store, y libros gratuitos como "The Swift Programming Language

SQL

Es un lenguaje de programación declarativo para gestionar bases de datos relacionales, que permite almacenar, consultar, manipular, actualizar y eliminar datos de forma organizada. Se utiliza en casi todas las empresas para manejar la información contenida en tablas y se considera un lenguaje esencial para la gestión de bases de datos, con aplicaciones en banca, telecomunicaciones y comercio electrónico

Base de datos relacional

Es un sistema que organiza datos en tablas compuestas por filas (registros) y columnas (atributos), utilizando SQL para gestionar y consultar la información, que está vinculada a través de claves entre las distintas tablas. Son fundamentales en programación para construir aplicaciones que requieren almacenamiento estructurado y relaciones complejas entre datos, como en sistemas de comercio electrónico, CRM y ERP.

MySQL

Es un sistema de gestión de bases de datos relacionales (SGBDR) de código abierto que utiliza el lenguaje SQL para crear, organizar y gestionar datos estructurados en tablas. Permite a los programadores interactuar con la base de datos mediante comandos estándar de SQL, siendo una opción popular para sistemas de gestión de contenido como WordPress y para el desarrollo web en lenguajes como PHP, Python y Java.

MongoDB

Es una base de datos NoSQL y orientada a documentos que almacena datos en formato flexible de tipo BSON (similar a JSON), en lugar de las tablas y filas de las bases de datos relacionales. Es útil para manejar datos estructurados, semiestructurados y no estructurados, y ofrece escalabilidad, flexibilidad de esquemas y un modelo de datos que se adapta directamente a los objetos en el código, facilitando la integración con diversos lenguajes de programación como Python, Java y JavaScript.

PostgreSQL

Es un sistema de base de datos relacional de objetos (ORDBMS) de código abierto que se utiliza para almacenar, gestionar y recuperar datos de forma fiable y eficiente. Es un componente fundamental en el desarrollo de aplicaciones, especialmente web, móviles y geoespaciales, gracias a su solidez, extensibilidad y soporte para SQL avanzado y datos no relacionales como JSON. Los desarrolladores lo usan para construir aplicaciones escalables al interactuar con él mediante código, a menudo desde diferentes lenguajes de programación.

Indice de base de datos

Es una estructura de datos que mejora la velocidad de búsqueda al crear una tabla de referencia de los valores de una o más columnas de una tabla, con punteros a las filas donde se encuentran esos datos. Permite a la base de datos encontrar rápidamente los datos sin tener que escanear toda la tabla, similar a cómo el índice de un libro ayuda a encontrar información.

Normalización

Es la transformación de las vistas de usuario complejas y del almacén de datos a un juego de estructuras de datos más pequeñas y estables. Además de ser más simples y estables, las estructuras de datos son más fáciles de mantener que otras estructuras de datos

Desnormalización

En programación y bases de datos es el proceso intencional de introducir redundancia de datos en una base de datos normalizada para mejorar el rendimiento de las consultas y la velocidad de lectura de datos, aunque esto incrementa el riesgo de inconsistencia. Se logra combinando tablas, agregando columnas o duplicando datos, y es útil en sistemas con un alto volumen de lecturas, como almacenes de datos y sistemas OLAP, donde la velocidad es más crítica que la eficiencia en las escrituras o la integridad absoluta de los datos

Transacción

Es un conjunto de una o más operaciones que deben tratarse como una única unidad lógica de trabajo, de modo que o se ejecutan todas o ninguna lo hace. Para asegurar la integridad de los datos, las transacciones deben cumplir las propiedades ACID: Atomicidad, Consistencia, Aislamiento y Durabilidad.

ACID

Es un acrónimo que describe las propiedades clave (Atomicidad, Consistencia, Aislamiento y Durabilidad) que garantizan la integridad y confiabilidad de las transacciones en las bases de datos. Estas propiedades aseguran que las operaciones se realicen correctamente, previniendo inconsistencias y garantizando que los datos se mantengan seguros y precisos, incluso ante fallos.

Dato

Es cualquier elemento de información (un número, texto, etc.) que un programa puede interpretar y manipular. Los tipos de datos (como enteros, flotantes o booleanos) clasifican esta información para definir qué valores puede almacenar una variable, qué operaciones se pueden realizar con ella y cuánto espacio ocupará en la memoria

Información

Es el resultado de procesar, organizar y estructurar datos para que adquieran un significado y valor. Se organiza en tipos de datos (numéricos, texto, booleanos, listas), que determinan qué valores puede almacenar una variable y las operaciones permitidas. Los programas manipulan estos tipos de datos para realizar cálculos, tomar decisiones y automatizar tareas.

Conocimiento

Incluyen una base de lógica y algoritmos, la sintaxis de lenguajes específicos (como Python o JavaScript), el manejo de estructuras de datos, el uso de control de versiones (Git), la capacidad de depurar código, y la comprensión de bases de datos y programación orientada a objetos. También se valoran el conocimiento general del ecosistema de desarrollo (frontend y backend), la habilidad para escribir código limpio y documentado, y la integración en equipos de trabajo mediante la comunicación efectiva.

Analítica descriptiva

Consiste en la recopilación, organización y presentación de datos pasados para comprender qué ha ocurrido, respondiendo a la pregunta "¿Qué pasó?". Se apoya en técnicas estadísticas como la media, mediana, desviación estándar y visualizaciones (gráficos de barras, circulares, etc.) para identificar patrones y tendencias, siendo un paso fundamental para el análisis de datos posterior.

Analitica predictiva

Son el proceso de utilizar datos para prever resultados futuros. El proceso utiliza análisis de datos, aprendizaje automático, inteligencia artificial y modelos estadísticos para detectar patrones que puedan predecir el comportamiento futuro.

Analitica prescriptiva

Es una disciplina avanzada de análisis de datos que no solo predice eventos futuros, sino que también recomienda las acciones óptimas a tomar para lograr un resultado deseado, empleando para ello el aprendizaje automático, la inteligencia artificial (IA), y técnicas de optimización. Su objetivo es responder a la pregunta "¿qué se debe hacer?", proporcionando decisiones proactivas y estrategias para aprovechar oportunidades o mitigar riesgos

Data mining

Es un proceso técnico, automático o semiautomático, que analiza grandes cantidades de información dispersa para darle sentido y convertirla en conocimiento.

Big Data

Se refiere al uso de lenguajes y herramientas para manejar y analizar conjuntos de datos masivos y complejos que no pueden ser procesados con sistemas tradicionales. Los programadores usan el Big Data para extraer información valiosa que ayude a las empresas a tomar mejores decisiones, personalizar servicios y predecir tendencias, utilizando tecnologías como procesamiento paralelo, aprendizaje automático e inteligencia artificial. Lenguajes como Python son fundamentales para esta tarea, gracias a su facilidad de uso y su gran ecosistema de bibliotecas y framework.

ETL

Son las siglas de "Extraer, Transformar y Cargar", un proceso fundamental de integración de datos que extrae información de diversas fuentes, la limpia y adapta según reglas de negocio específicas, y luego la carga en un repositorio centralizado, como un data warehouse o data lake, para análisis y toma de decisiones. Este proceso es crucial para la inteligencia empresarial, la generación de informes, el machine learning y la mejora de la eficiencia operativa

Data warehouse

Es un sistema centralizado que recopila, transforma y almacena grandes volúmenes de datos históricos de diversas fuentes para facilitar el análisis y la toma de decisiones estratégicas en una organización. Su objetivo es proveer una base de datos optimizada para consultas complejas y análisis, contrastando con las bases de datos operativas que gestionan transacciones diarias.

IA

Se refiere al uso de la inteligencia artificial para asistir, generar o mejorar código. Esto incluye herramientas que sugieren código en IDEs, generan código a partir de descripciones en lenguaje natural, y automatizan la depuración y optimización del software. Lenguajes como Python son fundamentales para desarrollar sistemas de IA, y herramientas como Gemini, TensorFlow y Tabnine son ejemplos de cómo se implementa la IA en el proceso de programación

Aprendizaje automático

En programación es un subcampo de la inteligencia artificial que permite a los sistemas aprender y mejorar de forma autónoma de grandes cantidades de datos, sin necesidad de ser programados explícitamente para cada tarea específica. Los programadores utilizan algoritmos que, al analizar datos, identifican patrones y relaciones para hacer predicciones o tomar decisiones, volviéndose más precisos con el tiempo a medida que son expuestos a más datos.

Red neuronal artificial

Es un modelo de inteligencia artificial en programación, inspirado en el cerebro humano, que aprende de datos para reconocer patrones y realizar predicciones. Se compone de capas de neuronas interconectadas con pesos que se ajustan durante el entrenamiento. Se utiliza en tareas como reconocimiento de imágenes y habla, traducción y diagnóstico médico, y se implementa comúnmente con lenguajes como Python y bibliotecas de machine learning.

Deep learning

Es un campo de la inteligencia artificial que utiliza redes neuronales artificiales con múltiples capas para modelar y entender patrones complejos en los datos. Programar en deep learning implica crear y entrenar estas redes neuronales con grandes conjuntos de datos, utilizando bibliotecas y framework especializados, para que la computadora aprenda autónomamente a realizar tareas como el reconocimiento de imágenes, la traducción de idiomas y la generación de texto.

Over fitting

El sobreajuste ocurre cuando un algoritmo se ajusta demasiado o incluso exactamente a sus datos de entrenamiento, lo que da como resultado un modelo que no puede hacer predicciones o conclusiones precisas a partir de ningún otro dato que no sea el de entrenamiento.

Procesamiento de lenguaje

Es el campo del Procesamiento del Lenguaje Natural (PLN o NLP por sus siglas en inglés), una subdisciplina de la Inteligencia Artificial que permite a las computadoras comprender, interpretar y generar el lenguaje humano escrito o hablado. Se combina la lingüística, la informática y el aprendizaje automático para que las máquinas puedan entender el significado completo y la intención detrás de las palabras, incluyendo el sentimiento y el contexto.

Visión por computadora

Es un campo de la inteligencia artificial que permite a las máquinas "ver", interpretar y comprender imágenes y videos de forma similar a los humanos. En programación, implica el uso de algoritmos y aprendizaje profundo, como las redes neuronales convolucionales, para extraer información visual, reconocer objetos y patrones, y tomar decisiones inteligentes, con el objetivo de aplicar esta capacidad en diversas áreas como la robótica, la medicina, la seguridad y los vehículos autónomos

Under fitting

El subajuste es un escenario en la ciencia de datos en el que un modelo de datos no puede capturar con precisión la relación entre las variables de entrada y salida, lo que genera una alta tasa de error tanto en el conjunto de entrenamiento como en los datos no vistos.

Modelo supervisado

Los modelos de aprendizaje supervisado se centran más en el aprendizaje de las relaciones entre los datos de entrada y salida. Por ejemplo, un modelo supervisado podría utilizarse para predecir los horarios de los vuelos en función de parámetros específicos, como las condiciones meteorológicas, el tráfico aeroportuario, las horas punta de vuelo, etc.

Hadoop

Es un framework (marco de trabajo) de código abierto, basado en Java, que permite almacenar y procesar de forma distribuida grandes volúmenes de datos (Big Data) en clústeres de ordenadores. Su modelo de programación se centra en dividir los datos en bloques, distribuirlos en el clúster y ejecutar tareas en paralelo mediante los procesos de MapReduce, lo que resulta en un procesamiento rápido y eficiente de conjuntos de datos masivos.

Modelo no supervisado

En programación, específicamente en el ámbito del aprendizaje automático (Machine Learning), es un algoritmo que aprende patrones y estructuras en un conjunto de datos sin etiquetas de salida predefinidas. A diferencia del aprendizaje supervisado, el modelo trabaja de forma independiente para descubrir relaciones intrínsecas en los datos, realizando tareas como la agrupación (clustering), la detección de anomalías o la reducción de la dimensionalidad

Spark

Se refiere principalmente a la programación con Apache Spark, un motor de procesamiento de datos de código abierto y en memoria para el análisis de big data, conocido por su velocidad y su API para Java, Scala, Python, R y SQL. También puede referirse al lenguaje de programación SPARK, un subconjunto seguro de Ada diseñado para sistemas de alta integridad.

clúster

Es un grupo de computadoras o nodos interconectados que trabajan juntos como un único sistema para ejecutar tareas paralelas, mejorar el rendimiento y proporcionar alta disponibilidad y tolerancia a fallos. Se utiliza en aplicaciones de alto rendimiento, como la renderización de imágenes o el análisis de datos, y en entornos de computación en la nube con orquestadores como Kubernetes.

Escalabilidad vertica

En programación (también conocida como "escalado hacia arriba" o "scale-up") consiste en aumentar los recursos de hardware de una única máquina existente, como su CPU, memoria (RAM) o almacenamiento, para que pueda manejar cargas de trabajo más grandes o una mayor actividad del usuario. Es una estrategia simple para mejorar el rendimiento sin cambiar el código ni la arquitectura del sistema, aunque tiene limitaciones y un riesgo de punto único de fallo.

Escalabilidad horizonta

En programación es la técnica de añadir más máquinas o servidores (nodos) a un sistema existente para distribuir la carga de trabajo y aumentar la capacidad, en lugar de potenciar un único servidor. Esto permite manejar mayores volúmenes de datos o tráfico, mejorar el rendimiento y aumentar la resiliencia, siendo ideal para aplicaciones distribuidas y elásticas.

Computación en la nube

Se refiere a la entrega de recursos informáticos como servidores, almacenamiento, bases de datos, redes y herramientas de desarrollo a través de Internet, en lugar de depender de la infraestructura física local. Permite a los programadores acceder bajo demanda a estos recursos, escalar sus aplicaciones de forma flexible y pagar solo por el uso, acelerando el desarrollo y la innovación.

IaaS

Se refiere a un modelo de computación en la nube que proporciona a los desarrolladores recursos de infraestructura virtualizada bajo demanda, como servidores, almacenamiento y redes, sobre los cuales pueden desplegar y ejecutar sus aplicaciones. En lugar de adquirir y mantener hardware físico, los programadores alquilan estos recursos a un proveedor externo, pagando solo por el uso, lo que permite escalar recursos fácilmente, reducir costos de infraestructura y agilizar el desarrollo al poder aprovisionar y desmantelar rápidamente entornos de prueba y desarrollo.

PaaS

Es un modelo de computación en la nube que ofrece un entorno completo para el desarrollo y la ejecución de aplicaciones, encargándose el proveedor de la infraestructura (hardware, software, redes, almacenamiento) para que los desarrolladores puedan centrarse únicamente en el código y la lógica de sus aplicaciones. Esto simplifica el proceso, acelera el desarrollo y reduce los costos asociados con la gestión de infraestructura local

SAAS

Se refiere a "Software como Servicio" (Software as a Service), un modelo de distribución de software basado en la nube donde el proveedor aloja las aplicaciones y las hace accesibles para los usuarios a través de internet en lugar de instalar el software localmente. Los usuarios pagan una tarifa periódica (como una suscripción) para acceder a la aplicación, que el proveedor se encarga de mantener, actualizar y gestionar, liberando a los usuarios de estas tareas

Serverless

Es un modelo de desarrollo en la nube donde el proveedor se encarga de la infraestructura y la administración de servidores, permitiendo a los desarrolladores enfocarse en el código de sus aplicaciones en forma de funciones pequeñas y eventos. El término se refiere a la abstracción de los servidores, no a su ausencia, y el código se ejecuta bajo demanda, con un modelo de pago por uso.

PSeInt

Es un software gratuito de código abierto para aprender a programar, enfocado en enseñar los fundamentos de la algoritmia a través de un pseudolenguaje en español. Es una herramienta educativa que permite a los usuarios escribir y ejecutar algoritmos en pseudocódigo, verlos como diagramas de flujo, y obtener ayuda para la corrección de errores, haciendo que el aprendizaje sea intuitivo y didáctico.

Metodología Agile

Es un enfoque iterativo de desarrollo de software que prioriza la flexibilidad, la colaboración y la entrega continua de valor para el cliente, dividiendo el proyecto en ciclos cortos (sprints) para adaptarse al cambio de manera rápida y eficiente, en contraste con los enfoques tradicionales como la cascada. Fomenta la retroalimentación constante, la participación del cliente y la adaptabilidad, resultando en productos de mayor calidad y mayor satisfacción del cliente.

Scrum

Es un marco de trabajo de la metodología ágil que los equipos de desarrollo de software utilizan para gestionar proyectos complejos mediante ciclos de trabajo corto e iterativo, llamados Sprints. Se basa en la transparencia, inspección y adaptación, permitiendo a los equipos auto organizarse, aprender de la experiencia y ajustarse a los cambios.

Kanban

Es un método ágil para visualizar y gestionar el flujo de trabajo mediante un tablero que muestra las tareas en diferentes etapas (como "por hacer", "en progreso" y "hecho"). Su objetivo es mejorar la eficiencia y la productividad al permitir que los equipos identifiquen cuellos de botella, limiten el trabajo en curso (WIP) y se adapten ágilmente a los cambios

Sprint

Es un período de tiempo fijo y corto, usualmente entre una y cuatro semanas, dentro de las metodologías ágiles como Scrum. Durante este ciclo, el equipo trabaja de forma colaborativa para desarrollar un conjunto específico de funcionalidades o un incremento de valor para el producto. Cada Sprint incluye etapas clave como la planificación, las reuniones diarias (Daily Stand-ups), el desarrollo, la revisión del trabajo y una retrospectiva para mejorar continuamente el proceso

Product Backlog

Es una lista dinámica y priorizada de elementos de trabajo, como características, correcciones de errores y mejoras, que deben realizarse para un producto. Es el artefacto principal en las metodologías ágiles como Scrum y es gestionado por el Product Owner, quien se asegura de que el Backlog guíe al equipo de desarrollo hacia los objetivos del producto.

Daily Standup

Es una reunión diaria y corta (generalmente 15 minutos), característica de las metodologías ágiles como Scrum, donde los miembros de un equipo de desarrollo se sincronizan sobre el progreso del proyecto, planean el día y se identifican obstáculos que impiden avanzar. El nombre "standup" proviene de la práctica de reunirse de pie para fomentar la brevedad y la concentración.

Product Owner

Es el responsable de maximizar el valor de un producto en metodologías ágiles como Scrum. Actúa como el principal punto de contacto entre los stakeholders (clientes, equipo de marketing, etc.) y el equipo de desarrollo, traduciendo las necesidades del negocio en requisitos claros y definiendo un backlog del producto priorizado. Su trabajo consiste en tener una visión clara del producto, gestionar su hoja de ruta y asegurar que el equipo construya lo correcto para entregar el mejor resultado posible

USER STORY

En programación es una descripción informal y breve de una funcionalidad de software, escrita desde la perspectiva del usuario final, que detalla lo que el usuario necesita y por qué lo necesita, generalmente siguiendo el formato: "Como [tipo de usuario], quiero [acción] para [beneficio]". Las user stories son un componente clave del desarrollo ágil, ya que facilitan la comunicación, la colaboración entre equipos y clientes, y permiten dividir el trabajo en entregas pequeñas y de valor rápido.

Ciberseguridad

Consiste en utilizar lenguajes como Python, Bash, y JavaScript para crear soluciones que protejan sistemas, detecten amenazas, automatizar procesos de seguridad, y analizar vulnerabilidades. Esta aplicación se traduce en el desarrollo de firewalls, herramientas de análisis forense, scripts para la respuesta a incidentes, y la protección de aplicaciones web.

Firewall

Es una herramienta de seguridad de red, ya sea hardware o software, que monitorea y filtra el tráfico de datos entrante y saliente de un sistema o red según reglas predeterminadas, actuando como una barrera para prevenir accesos no autorizados y proteger contra amenazas. Los programadores configuran estos firewalls para controlar qué aplicaciones pueden enviar y recibir información, y a qué destinos pueden comunicarse, mejorando la seguridad de las aplicaciones y los datos.

Malware

Es un programa o código escrito intencionalmente para causar daño, robar datos o obtener acceso no autorizado a sistemas informáticos, redes o usuarios. Los delincuentes cibernéticos utilizan el malware para fines como el robo de información personal o financiera, la extorsión mediante ransomware o la interrupción de servicios. Ejemplos comunes de malware incluyen virus, gusanos, ransomware y spyware, y se propaga a menudo a través de archivos o enlaces maliciosos que requieren la interacción del usuario.

Burn Down Chart

Diagrama de Trabajo Pendiente es una herramienta de gestión de proyectos ágil que muestra el trabajo restante por hacer en relación con el tiempo. Su objetivo es visualizar y predecir el progreso de un proyecto o sprint, ayudando a los equipos a identificar desviaciones y tomar acciones correctivas para cumplir los plazos y objetivos.

Phishing

En programación no se refiere a una técnica de programación en sí, sino a un ciberataque de suplantación de identidad que usa la programación y la ingeniería social para engañar a usuarios y programadores, robando así información sensible como contraseñas, datos bancarios y otros datos personales o de la organización. Los atacantes usan técnicas como enlaces falsos, correos electrónicos o mensajes que parecen legítimos, o la creación de sitios web fraudulentos que imitan a los oficiales.

Ransomware

Se refiere a un tipo de software malicioso (malware) que cifra (bloquea) archivos o sistemas para exigir un pago de rescate al usuario a cambio de la clave de descifrado, inutilizando el dispositivo hasta que se realiza el pago.

Antivirus

Es un software diseñado para detectar, prevenir y eliminar programas maliciosos (malware) como virus, gusanos, troyanos y ransomware que podrían interferir con el código y el funcionamiento de un sistema. Los desarrolladores utilizan el antivirus para proteger sus propios sistemas y el de sus usuarios, asegurando la integridad de las aplicaciones y el bloqueo de amenazas que podrían corromper o robar datos

Encriptación

Es el proceso de convertir datos legibles (texto plano) en un formato ilegible (texto cifrado o encriptado) usando una clave criptográfica, protegiendo así la información de accesos no autorizados. El proceso inverso, el descifrado, requiere la clave correcta para restaurar los datos a su forma original. Los principales tipos son la encriptación simétrica y asimétrica, cada una con diferentes algoritmos como el AES, utilizadas para asegurar la privacidad y la integridad de los datos en reposo o en tránsito.

CLave publica

Es un concepto de la criptografía asimétrica que se utiliza para cifrar información o verificar firmas digitales, mientras que una clave privada correspondiente es necesaria para descifrarlas o crearlas. La clave pública se comparte libremente y es utilizado por cualquiera para encriptar un mensaje destinado a un usuario específico, o para confirmar que una firma digital fue creada con la clave privada de esa persona.

Clave privada

Es un secreto criptográfico que, junto a una clave pública, se usa en la criptografía asimétrica para descifrar mensajes cifrados por la clave pública correspondiente y para crear firmas digitales, garantizando la confidencialidad e integridad de los datos, o para autenticar a un usuario. En contraste, la palabra clave prívate en lenguajes como Java o C# es un modificador de acceso que restringe el acceso a un miembro (variable, método o clase) solo a la clase donde se declara.

IDE

Es una aplicación de software que une todas las herramientas necesarias para el desarrollo de programas en una única interfaz, mejorando la eficiencia y productividad del programador al ofrecer un editor de código con resaltado de sintaxis, herramientas para automatizar la compilación del código, un depurador para encontrar errores, y otras funciones como autocompletado de código y gestión de proyectos, todo en un solo lugar.

Git

Es un sistema de control de versiones distribuido que permite gestionar cambios en el código fuente. Facilita la colaboración entre equipos de desarrollo al mantener un historial de modificaciones. Cada copia del repositorio contiene la historia completa, lo que lo hace muy robusto. Es ampliamente usado en proyectos de software modernos.

Github

Es una plataforma basada en Git para alojar repositorios en la nube .Ofrece herramientas de colaboración como issues, pull requests y proyectos. Permite integrar flujos de trabajo de CI/CD y automatización. Es una de las comunidades más grandes para desarrolladores.

Gitlab

Es una plataforma de desarrollo colaborativo que también usa Git como base.Se centra en la integración completa del ciclo DevOps en un solo lugar. Permite gestionar repositorios, CI/CD, despliegues y seguridad. Es una alternativa fuerte a GitHub y Bitbucket.

Bitbucket

Es un servicio de alojamiento de repositorios Git y Mercurial. Ofrecido por Atlassian, se integra con Jira y Trello. Facilita la gestión de ramas y equipos de trabajo. Se utiliza mucho en entornos empresariales.

CI/CD

Significa Integración Continua y Despliegue Continuo. Permite que el código se pruebe y se despliegue automáticamente tras cada cambio. Asegura mayor calidad y rapidez en el ciclo de desarrollo. Es una práctica clave en metodologías ágiles y DevOps.

Docker

Es una plataforma que permite empaquetar aplicaciones en contenedores. Los contenedores garantizan que la aplicación funcione igual en cualquier entorno. Reduce problemas de compatibilidad entre entornos de desarrollo y producción.Es ampliamente usado en despliegues modernos de software

Kubernetes

Es un orquestador de contenedores creado por Google. Permite gestionar aplicaciones distribuidas en múltiples contenedores. Automatiza despliegues, escalado y balanceo de cargas. Es estándar en infraestructura en la nube.

Jenkins

Es una herramienta de automatización para CI/CD. Permite construir, probar e implementar software de forma automática. Cuenta con cientos de plugins para integrarse con otras herramientas. Es muy usado en entornos DevOps.

Visual studio code

Es un editor de código fuente desarrollado por Microsoft. Soporta múltiples lenguajes de programación mediante extensiones. Incluye depuración, control de versiones y terminal integrada. Es uno de los editores más populares en la actualidad.

Testing de software

Es el proceso de verificar y validar un sistema o aplicación. Busca asegurar que cumple con los requisitos establecidos. Permite detectar defectos antes de que lleguen al usuario final. Es esencial para garantizar la calidad del software.

Pruebas unitarias

Son pruebas que verifican el funcionamiento de unidades pequeñas del código. Generalmente se aplican a funciones o métodos individuales. Permiten detectar errores rápidamente en fases tempranas. Son clave para mantener un software estable y confiable.

Pruebas de integración

Verifican que diferentes módulos del sistema trabajen correctamente juntos. Aseguran la comunicación entre componentes de software. Detectan errores en la interacción de subsistemas. Son el siguiente paso después de las pruebas unitarias.

Pruebas funcionales

Evalúan si el software cumple con los requisitos funcionales. Se basan en la especificación del sistema. Simulan el uso real por parte de un usuario. Son fundamentales para validar la lógica del sistema.

Pruebas de rendimiento

Miden cómo responde un sistema bajo ciertas condiciones de carga. Evalúan tiempos de respuesta, consumo de recursos y estabilidad. Se usan para asegurar que el software pueda escalar. Son comunes en aplicaciones web de gran tráfico.

Pruebas de seguridad

Verifican la resistencia del software frente a ataques y vulnerabilidades. Analizan la protección de datos sensibles. Detectan posibles fallas de autenticación o accesos indebidos. Son claves en aplicaciones financieras o gubernamentales.

Pruebas de automatización

Son pruebas realizadas por scripts o herramientas en lugar de humanos. Aceleran el ciclo de pruebas en proyectos grandes. Permiten ejecutar escenarios repetitivos de manera eficiente. Se integran con CI/CD para mayor eficacia.

Tdd (Test driven development)

Es una metodología de desarrollo guiado por pruebas. Se escribe primero la prueba, luego el código que la hace pasar. Fomenta un diseño más claro y menos errores. Es muy usado en entornos ágiles.

Mocking

Es una técnica en pruebas unitarias. Consiste en simular objetos o dependencias del sistema. Permite probar una unidad sin necesidad de usar recursos externos reales. Facilita aislar el código bajo prueba.

Bug

Es un error o defecto en un programa informático. Puede provocar resultados inesperados o fallos en el sistema. Generalmente se detecta durante el testing o uso del software. La corrección de bugs es parte esencial del mantenimiento.

Patrón de diseño

Es una solución probada a un problema común en desarrollo de software. Se utilizan para estructurar el código de manera más eficiente. Aumentan la reutilización y mantenibilidad del software. Existen patrones creacionales, estructurales y de comportamiento.

Mvc

Es un patrón de arquitectura de software. Divide la aplicación en tres capas: Modelo, Vista y Controlador. El Modelo maneja datos, la Vista la interfaz y el Controlador la lógica. Facilita la organización y mantenimiento de proyectos grandes.

Microservicios

Es un estilo de arquitectura de software basado en pequeños servicios. Cada servicio es independiente y cumple una función específica. Permite escalar aplicaciones de manera más flexible. Se integra mediante APIs o mensajería.

Monolito

Es un modelo de arquitectura donde toda la aplicación es un solo bloque. El código, las funciones y la lógica están altamente integrados. Suele ser más simple de implementar, pero difícil de escalar. Es común en aplicaciones antiguas o pequeñas.

Patrón singleton

Es un patrón de diseño creacional. Garantiza que una clase tenga una única instancia en todo el sistema. Se usa para gestionar recursos compartidos como conexiones a bases de datos. Debe aplicarse con cuidado para no generar dependencias rígidas.

Patrón Factory

Es un patrón de diseño creacional que delega la creación de objetos a una clase especializada. Permite instanciar objetos sin exponer la lógica de creación al cliente. Favorece la reutilización de código y el principio de bajo acoplamiento.Es muy usado en framework y librerías de software.

Patrón Observer

Es un patrón de diseño de comportamiento. Define una relación uno-a-muchos entre objetos. Cuando un objeto cambia, notifica automáticamente a sus observadores. Se aplica en interfaces gráficas y sistemas de eventos.

Patrón Decorator

Es un patrón estructural que permite añadir funcionalidades a un objeto dinámicamente. Evita modificar directamente la clase original. Se logra envolviendo el objeto en una clase decoradora. Es común en librerías gráficas y de UI.

Patrón Adapter

Es un patrón estructural que actúa como traductor entre dos interfaces incompatibles. Permite que clases con diferentes interfaces trabajen juntas. Se implementa creando una capa intermedia de adaptación. Es muy usado en integración de sistemas.

Patrón Proxy

Es un patrón estructural que provee un sustituto de otro objeto. Controla el acceso al objeto real, añadiendo seguridad o caché. El cliente interactúa con el proxy en lugar del objeto original. Es útil para optimizar recursos en sistemas complejos.

Data Lake

Es un repositorio centralizado que almacena datos en bruto. Acepta información estructurada, semiestructurada y no estructurada. Permite un análisis flexible para Big Data. Es común en entornos de analítica avanzada.

Data Mart

Es un subconjunto de un Data Warehouse. Se centra en un área específica como ventas o marketing. Facilita consultas rápidas para usuarios de negocio. Es más pequeño y específico que un Data Lake.

Business Intelligence

Es el conjunto de herramientas y procesos para analizar datos de negocio. Permite transformar datos en información útil para la toma de decisiones. Incluye dashboards, reportes y análisis de tendencias. Es clave en la estrategia empresarial moderna.

Kpi

Es un indicador de rendimiento clave. Permite medir el éxito de un proceso o estrategia. Debe ser específico, medible y relevante. Se usa en gestión empresarial y análisis de desempeño.

Dashboard

Es una interfaz visual que muestra indicadores clave y métricas. Facilita la interpretación rápida de información crítica. Se compone de gráficos, tablas y alertas. Es esencial en Business Intelligence.

Data Governance

Es el conjunto de políticas para gestionar los datos en una organización. Busca garantizar calidad, seguridad y accesibilidad de la información. Define roles, procesos y responsabilidades. Es vital en entornos con grandes volúmenes de datos.

Data Steward

Es el responsable de la administración de los datos. Supervisa la calidad y consistencia de la información. Asegura el cumplimiento de las normas de Data Governance. Su rol es clave en organizaciones basadas en datos..

Data Quality

Se refiere a la precisión, consistencia y confiabilidad de los datos. Un buen nivel de calidad asegura mejores decisiones. Se mide en dimensiones como integridad, exactitud y actualidad. Es fundamental para proyectos de análisis.

Metadata

Es información que describe a otros datos. Ejemplos: autor, fecha de creación, tamaño de archivo. Facilita la organización y búsqueda de información. Es clave en bases de datos y bibliotecas digitales.

Olap

Es una tecnología para analizar datos multidimensionales. Permite hacer consultas complejas de manera rápida. Se usa en inteligencia de negocios y análisis financiero. Facilita la creación de cubos de datos.

Variable

Es un espacio en memoria que almacena un valor temporal. Su contenido puede cambiar durante la ejecución de un programa. En estadística representa una característica a medir. Es fundamental en programación y análisis de datos.

Media Aritmética

Es el promedio de un conjunto de valores. Se obtiene sumando los datos y dividiendo entre la cantidad. Es una medida de tendencia central muy usada. Ayuda a resumir grandes volúmenes de información.

Mediana

Es el valor central de un conjunto de datos ordenados. Divide la muestra en dos partes iguales. Es menos sensible a valores extremos que la media. Se utiliza mucho en estadística descriptiva.

Moda

Es el valor que más veces se repite en un conjunto de datos. Puede haber una o varias modas en una distribución. Es útil en variables categóricas o discretas. Complementa otras medidas estadísticas.

Desviación Estándar

Mide la dispersión de los datos respecto a la media. Valores bajos indican poca variabilidad. Valores altos reflejan gran dispersión de datos. Es fundamental en análisis estadístico.

Varianza

Es el promedio de las desviaciones cuadráticas respecto a la media. Indica la dispersión de los datos en un conjunto. Se mide en unidades al cuadrado.Es la base para calcular la desviación estándar.

Correlación

Mide la relación entre dos variables. Su valor va de -1 a +1. Un valor cercano a 1 indica relación positiva fuerte. Es muy usado en análisis estadístico y predicciones.

Regresión Lineal

Es una técnica estadística para modelar la relación entre variables. Predice una variable dependiente a partir de una independiente. Se representa con una línea recta ajustada a los datos. Es base en modelos predictivos.

Distribución Normal

Es una distribución de probabilidad simétrica en forma de campana. La mayoría de los valores se concentran en la media. Es muy usada en estadística y ciencias sociales. Representa fenómenos naturales y sociales.

Hipótesis Nula

Es una afirmación inicial en un análisis estadístico. Generalmente sostiene que no hay relación o efecto. Se contrasta con la hipótesis alternativa. Es clave en pruebas de significancia.

Hardware

Son los componentes físicos de un sistema informático. Incluye CPU, memoria, discos, teclado y monitor. Es esencial para ejecutar software. Es la parte tangible de la informática.

Software

Es el conjunto de programas y aplicaciones de un sistema. Permite ejecutar tareas y aprovechar el hardware. Se clasifica en sistema, aplicación y utilitarios. Es la parte intangible de la informática.

Sistema Operativo

Es el software base que gestiona hardware y aplicaciones. Controla procesos, memoria y dispositivos. Ejemplos: Windows, Linux, macOS. Es fundamental para usar un computador.

Procesador

Es la unidad central de procesamiento (CPU). Ejecuta instrucciones y operaciones lógicas. Determina la velocidad de un computador. Es considerado el cerebro del sistema.

Memoria Ram

Es la memoria de acceso aleatorio del computador. Almacena datos e instrucciones temporales. Permite ejecutar programas de manera eficiente. Se borra al apagar el equipo.

Almacenamiento SSD

Es un dispositivo de almacenamiento en estado sólido. Más rápido que los discos duros tradicionales (HDD). Usa memoria flash para guardar datos. Mejora el rendimiento de sistemas modernos

Virtualización

Es la creación de versiones virtuales de recursos físicos. Permite ejecutar varios sistemas operativos en un mismo hardware. Se usa en servidores, almacenamiento y redes. Es clave en entornos de nube.

Máquina Virtual

Es una emulación de un sistema operativo completo. Funciona sobre un host físico mediante virtualización. Permite ejecutar múltiples entornos en un mismo equipo. Aísla procesos para mayor seguridad y flexibilidad.

Servidor

Es un sistema que provee servicios a otros dispositivos. Puede manejar aplicaciones, bases de datos o archivos. Funciona de manera centralizada en redes. Es esencial en internet y empresas.

Cliente

Es un dispositivo o software que consume servicios de un servidor. Ejemplos: navegadores, apps o computadoras conectadas. Se comunica mediante protocolos de red. Es la contraparte del modelo cliente-servidor.

Blockchain

Es una tecnología de registro distribuido y seguro. Permite almacenar transacciones en bloques encadenados. Es descentralizada y resistente a fraudes. Es la base de criptomonedas como Bitcoin.

Criptomoneda

Es una moneda digital basada en criptografía. No depende de bancos centrales ni gobiernos. Se intercambia en redes descentralizadas blockchain. Ejemplos: Bitcoin, Ethereum.
